# Tech Insight Platform — 설계 문서 (v0.1)

> 기술 트렌드 인텔리전스 + AI 분석 챗봇 웹 서비스
> 작성일: 2026-06-02 · 대상: 소규모(나/팀) 내부 사용

---

## 1. 무엇을 만드는가

3가지를 하나로 묶은 서비스:

1. **지식베이스** — 논문(정보과학회지 등) + 해외 블로그 + 뉴스를 모아 검색 가능한 형태로 저장
2. **자동 수집** — 새 논문/블로그/뉴스가 나오면 자동으로 추가·분석
3. **분석 챗봇** — 뉴스나 질문을 입력하면, *수집한 데이터를 근거로* 파급효과·인사이트를 대화형으로 제시 (RAG 방식)

핵심 원칙: **"Claude가 일반론이 아니라, 우리가 모은 자료에 근거해서 답한다"** (RAG).

## 2. 전체 아키텍처

```
┌─────────────┐   ┌──────────────────┐   ┌────────────────────┐
│  수집 계층   │   │   처리/저장 계층    │   │     질의/대화 계층    │
├─────────────┤   ├──────────────────┤   ├────────────────────┤
│ PDF 논문     │   │ 텍스트 추출         │   │ 사용자 질문/뉴스 입력 │
│ 블로그 RSS   │ → │ 청크 분할          │ → │      ↓             │
│ arXiv API   │   │ 임베딩(벡터화)      │   │ 관련 문서 검색       │
│ 뉴스        │   │      ↓            │   │ (벡터 유사도)        │
└─────────────┘   │ [관계형 DB] 메타    │   │      ↓             │
                  │ [벡터 DB]  본문벡터  │ → │ Claude API 호출      │
                  └──────────────────┘   │ (근거문서 + 질문)     │
                                         │      ↓             │
                                         │ 근거 기반 답변 + 출처 │
                                         └────────────────────┘
```

## 3. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| 백엔드 | **Django + Django REST Framework** | 이미 설치됨. 관리자페이지/ORM/API 일체형 |
| LLM | **Claude API** (claude-opus / sonnet) | 한국어·장문 분석 강점. 키 보유 |
| 임베딩 | Claude 또는 오픈소스(예: BGE-m3 다국어) | 한국어 문서 → 다국어 임베딩 권장 |
| 관계형 DB | 초기 SQLite → 운영 PostgreSQL | 메타데이터·문서·키워드 |
| 벡터 검색 | 초기 **Chroma**(로컬, 쉬움) → 운영 **pgvector** | 문서 의미 검색 |
| 문서 처리 | pdfplumber(PDF), trafilatura(블로그), feedparser(RSS) | |
| 수집 자동화 | Django management command + 스케줄러(cron) | "새 글 자동 추가" |
| 프론트 | 초기 Django 템플릿 → 챗봇 UI는 추후 React 검토 | 단계적 |

## 4. 데이터 모델 (초안)

```
Source(출처)
  - name            예: "정보과학회지", "OpenAI Blog", "arXiv cs.AI"
  - type            paper | blog | news
  - url, rss_url    수집 경로
  - is_active

Document(문서) ★ 핵심
  - source (FK)
  - title, authors, affiliations
  - published_date  (예: 2026-05)
  - raw_text        추출된 본문 전체
  - summary         AI 생성 요약
  - url / file_path 원본 위치
  - status          collected | extracted | embedded | analyzed
  - created_at

Chunk(본문 조각)   ← RAG 검색 단위
  - document (FK)
  - text            300~800 토큰 단위 조각
  - embedding_id    벡터 DB 내 ID

Keyword(키워드)
  - name, category
  - lifecycle       성장 | 신규 | 쇠퇴 | 변동
  (Document와 M2M)

ChatSession / ChatMessage  ← 대화 이력
  - role(user/assistant), content, cited_documents
```

> 기존 `test/index.html`의 하드코딩 `DATA` 객체가 곧 이 테이블들의 한 스냅샷에 해당. 1단계에서 이걸 DB로 이관.

## 5. RAG 챗봇 동작 흐름

1. 사용자: *"이 뉴스의 파급효과는?"* + 뉴스 본문 입력
2. 입력 텍스트를 임베딩 → 벡터 DB에서 **유사 문서 top-K** 검색
3. 검색된 문서 + 질문을 Claude API 프롬프트로 구성
   - 시스템: "너는 기술 트렌드 분석가다. 아래 근거 문서만 사용해 답하라"
   - 컨텍스트: 근거 문서들 (출처 표시)
   - 질문: 사용자 입력
4. Claude 응답 → **출처(어떤 논문/블로그 기반인지)와 함께** 표시
5. 대화 이력 저장 (후속 질문 맥락 유지)

**비용 절감 포인트:** 근거 문서는 prompt caching으로 캐싱, 임베딩은 오픈소스 모델 사용 가능.

## 6. 자동 수집 흐름

```
정기 실행(매일/매주) → 각 Source의 RSS/API 확인 →
신규 항목 발견 → 본문 추출 → 청크 분할 → 임베딩 →
DB + 벡터DB 저장 → (선택) 자동 요약/키워드 태깅
```

- arXiv: 공식 API 있음 (카테고리별 신규 논문)
- 해외 블로그: RSS 피드 우선, 없으면 크롤링
- 뉴스: RSS 또는 수동 입력

```

├── data_source/          # 원본 파일 (현행 유지)

│   ├── models.py         # 위 데이터 모델

│   ├── pipeline/         # 추출→청크→임베딩
│   ├── rag/              # 검색 + Claude 호출
│   ├── views.py / api.py # 웹·API
│   └── management/commands/  # collect, embed 등 배치 명령
├── dashboard/            # 인사이트 대시보드 (기존 index.html 동적화)
└── templates/            # 챗봇·대시보드 UI
```

## 8-1. 대시보드 통합 단계 (test/index.html → DB 연동)

- **레벨 1**: index.html을 Django 페이지로 서빙만 (데이터 하드코딩 유지)

  - 하드코딩 유지: 키워드 히트맵/라이프사이클, 특집호 주제, 글로벌 시차 (분석 필요)
